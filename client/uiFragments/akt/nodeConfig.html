Build & Setup an x86 Cluster
For non-programmers/non-builders we make it as simple as possible to build an x86 (AMD | Intel CPU) cluster to host Akash.
Instructions/Tutorial:
Step 1: Go Shopping +
  • Buy 3 or 4 UDOO Bolt Gear for ~$450/each UDOO Bolt Store
  • Each UDOO Bolt supports up to 32GB RAM (2400MHz, 260 pins). I buy a single 8GB stick per machine for ~$35 Amazon
  • The UDOO Bolt supports NVMe storage. I buy a single m.2 512GB B-key storage device per machine for ~$60 eBay
  • You need an Ethernet hub for fast and reliable internet connectivity (do not use wifi!) Amazon
  • Dont forget some short (6-12 inch) ethernet cables for internet Amazon
  • Buy a single USB Thumb drive (~16-32GB) for flashing Ubuntu Server OS onto each machine.
  • Alternately: Feel free to build any kind of PC you choose, just make sure it has a decent quad-core CPU (AMD|Intel only, NOT ARM or Raspberry Pi as Akash does not support ARM at this time), at least 2GB RAM per CPU Core, and at least 512GB NVMe storage.
    Note: We already built HandyHost with ARM/Raspberry Pi support so once it is supported you will be able to use it.
  • Link to our build spreadsheet on Google Drive
Step 2: Assemble +
  • Part 1: Assembling the Hardware: YouTube
  • Part 2: Setting up the Software: YouTube
Step 3: Create your USB ThumbDrive Auto-Installer +
    We made configuring the Akash nodes as simple as possible. We have created a custom Ubuntu Server ThumbDrive installer that you can plug into your new machines and have them auto-configured and setup for access for the Akash Cluster.
  1. Plug in your USB ThumbDrive to the machine running HandyHost.
  2. Use the utility below to detect the ThumbDrive and burn Ubuntu Server onto the ThumbDrive.
  3. After the Image is complete, you will configure the akash servers with the ThumbDrive. Note: The HandyHost masternode (the machine burning the USB) IS NOT the machine you want to do akash hosting on.
  4. When installing the thumbdrive OS on the nodes, first make sure your akash node is powered off and plugged into the internet. One at a time, plug the thumbdrive into any of your Akash servers and power the unit on.
    Note: The bootable thumbdrive will take over the server, erase its largest disk, install a secured version of Linux and setup communications for Akash cluster operations. If you have important data on the akash server please back it up.
  5. In about 5-10 minutes, the Thumbdrive installer takes care of the entire installation/configuration of the OS and will notify you within the HandyHost app when the new node is online, so leave this page open.
  6. Once the new node shows as online in your nodes list, you can unplug the ThumbDrive and if needed, plug the ThumbDrive into the next machine, repeat the previous step, until all your nodes are added.

Build & Setup a Raspberry Pi Cluster
For non-programmers/non-builders we make it as simple as possible to build a Raspberry Pi cluster to host Akash.
Instructions/Tutorial:
Step 1: Go Shopping +
  • Buy 3 or 4 Raspberry Pi 4b, 8GB [link]
  • Want less wires/adapters? Buy PoE (power-over-ethernet) hats per each Pi to get power over the hard-line internet cables [link]
  • Active Cooling is essential. Buy a Raspberry Pi Cluster case with a fan [link]
  • DO NOT USE MICROSD STORAGE. It is extremely slow and prone to fail. Instead buy fast/reliable NVMe Solid State Drives, we recommend a 512GB per Pi [link]
  • Since the Pi NVMe plugs in via USB3, you'll need an enclosure. This one is sleek and has a built in heat-sync (they get hot FYI) [link]
  • Dont forget some short (6-inch) ethernet cables for internet+power [link]
  • You need a Powered Ethernet hub for power + fast internet connectivity (do not use wifi) [link]
  • Buy a single MicroSD card and reader. The storage can be miniscule (8GB) because we only use it one time to instruct the Raspberry Pi to boot from the NVMe disk. [link]
  • Buy some standoffs to securely mount & stack our Raspberry Pi's [link]
Step 2: Configure Storage Drives +
  1. Download Raspberry Pi Imager. We use it to flash the Operating System onto the NVMe drives.
  2. Before we do anything fun, we need to tell each Raspberry Pi to boot from USB disk (NVMe).
    • Plug in your single MicroSD card/reader and Open Raspberry Pi Imager
    • Click OS and choose "Misc Utility Images" >> Bootloader >> USB Boot
    • Click "Choose Storage" and choose your MicroSD card
    • Click Write to burn the USB Boot loader onto the SD card
    • Once finished, one Raspberry Pi at a time, insert the SD card into the Raspberry Pi card slot.
    • Plug in the Raspberry Pi to power and notice the green LED is solid for about 5 seconds. After it starts to blink steadily it means we finished adding USB Boot. You can now unplug the Raspberry Pi, remove the SD card and move onto the next Raspberry Pi, repeating this step until finished.
  3. Now we burn Ubuntu onto our NVMe drives.
    • Assemble the NVMe drive, plug in the USB and open Raspberry Pi Imager.
    • Click OS and choose "Other General Purpose OS" >> Ubuntu >> Ubuntu Server 21.04 64-bit. It is crucial you choose the 64-bit Server Ubuntu
    • Click "Choose Storage" and locate your NVMe drive from the list of attached devices. It should be labeled NVMe and have a similar capacity (ie: 512GB)
    • Click Write and confirm you erase the disk.
  4. After Ubuntu is on the NVMe drive, the OS will un-mount the USB. For the next step we want to unplug it, then plug it back into the machine running HandyHost.
    Use the form below to detect and auto-configure your NVMe drive.

Welcome To Akash
Using HandyHost you can rent servers on Akash Marketplace for passive income generation.
We designed this tool to make it easy for non-programmers to point and click their way thru making a hardware cluster (a group of servers) to host Akash servers.
Step 1: Choose your Hardware
We have built and tested recommended hardware guides you can follow that are affordable both from a cost perspective as well as an electricity bill.
However that should not limit your hardware choices. You can choose to bring your own x86 (Intel or AMD) based computers into HandyHost and host Akash servers on them with our simple tools.
Step 2: Install the OS
Using a USB ThumbDrive, you will use HandyHost to create a Linux installer that will automatically setup the latest Ubuntu Server on your Akash node hardware, setup secure access for the HandyHost/Akash application and configure IP access on your local network for Akash to use.

Note: You must provide a wildcard-enabled domain name via services like: Godaddy, AWS Route53, noip.com or others. The wildcard domain should be configured to resolve to your global IP address. Get My Global IP
A region name is a freeform name you make up for your geographic region. Common regions: westcoast, eastcoast, midwest, us-west, etc. Note that region name will not stop you from winning or losing contracts, it helps renters more than anything.
A tier name is a freeform name you make up for your machine tiers. It is recommended (and default) to use "community", however you may opt for some other value in the future to describe the tier of hardware you operate.
We generate a random name for your cluster, feel free to change it to anything you would like to be known as.

Step 6: Open Provider Ports
You must open ports 80 and 30000-32767 to your IP in your internet router Port Forwarding. The machine runs the Ingress Controller which is responsible to route all internet traffic to nodes in your Akash Kubernetes Cluster.
You must also open the Akash Provider port 8443 for TCP traffic for IP: in your network router port forwarding.
How to Setup Port Forwarding on Your Router
Final Step: Build Kubernetes Cluster
Akash uses a software called Kubernetes to manage your cluster of servers. This step will take roughly 15-20 minutes to complete. Once complete, you can start up your provider and start hosting!
A few things to make sure of in the meantime:
  1. Ensure your wildcard DNS entry is setup
  2. Ensure that the ports listed above are open in your router. Also make sure that the IP addresses set by your router do not change on a router or system reboot.
  3. If the Kubernetes cluster fails to build and there are errors about reaching your nodes thru ssh you need to ensure that your router resolves local DNS names.
  4. You should have some $AKT in your wallet. You will need at least a 5AKT deposit per lease, and some AKT for fees when you bid.
