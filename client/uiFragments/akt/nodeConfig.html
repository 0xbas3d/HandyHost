Build & Setup a Raspberry Pi Cluster
For non-programmers/non-builders we make it as simple as possible to build a Raspberry Pi cluster to host Akash.
Instructions/Tutorial:
Step 1: Go Shopping +
  • Buy 3 or 4 Raspberry Pi 4b, 8GB [link]
  • Want less wires/adapters? Buy PoE (power-over-ethernet) hats per each Pi to get power over the hard-line internet cables [link]
  • Active Cooling is essential. Buy a Raspberry Pi Cluster case with a fan [link]
  • DO NOT USE MICROSD STORAGE. It is extremely slow and prone to fail. Instead buy fast/reliable NVMe Solid State Drives, we recommend a 512GB per Pi [link]
  • Since the Pi NVMe plugs in via USB3, you'll need an enclosure. This one is sleek and has a built in heat-sync (they get hot FYI) [link]
  • Dont forget some short (6-inch) ethernet cables for internet+power [link]
  • You need a Powered Ethernet hub for power + fast internet connectivity (do not use wifi) [link]
  • Buy a single MicroSD card and reader. The storage can be miniscule (8GB) because we only use it one time to instruct the Raspberry Pi to boot from the NVMe disk. [link]
  • Buy some standoffs to securely mount & stack our Raspberry Pi's [link]
Step 2: Configure Storage Drives +
  1. Download Raspberry Pi Imager. We use it to flash the Operating System onto the NVMe drives.
  2. Before we do anything fun, we need to tell each Raspberry Pi to boot from USB disk (NVMe).
    • Plug in your single MicroSD card/reader and Open Raspberry Pi Imager
    • Click OS and choose "Misc Utility Images" >> Bootloader >> USB Boot
    • Click "Choose Storage" and choose your MicroSD card
    • Click Write to burn the USB Boot loader onto the SD card
    • Once finished, one Raspberry Pi at a time, insert the SD card into the Raspberry Pi card slot.
    • Plug in the Raspberry Pi to power and notice the green LED is solid for about 5 seconds. After it starts to blink steadily it means we finished adding USB Boot. You can now unplug the Raspberry Pi, remove the SD card and move onto the next Raspberry Pi, repeating this step until finished.
  3. Now we burn Ubuntu onto our NVMe drives.
    • Assemble the NVMe drive, plug in the USB and open Raspberry Pi Imager.
    • Click OS and choose "Other General Purpose OS" >> Ubuntu >> Ubuntu Server 21.04 64-bit. It is crucial you choose the 64-bit Server Ubuntu
    • Click "Choose Storage" and locate your NVMe drive from the list of attached devices. It should be labeled NVMe and have a similar capacity (ie: 512GB)
    • Click Write and confirm you erase the disk.
  4. After Ubuntu is on the NVMe drive, the OS will un-mount the USB. For the next step we want to unplug it, then plug it back into the machine running HandyHost.
    Use the form below to detect and auto-configure your NVMe drive.
